# LLM Model Evolution Replication Package
Replication Package for the thesis:  
**"Do LLMs Improve in Modelling Tasks With Their Evolution? An Empirical Study"**

## Overview of the Package

### datasets
This folder contains the datasets used during the evaluation process.
- `synthetic_few_shot_dataset.jsonl` contains the original Synthetic dataset.
- `synthetic_plus_custom.jsonl` contains the extended synthetic dataset with additional custom samples.
- `repairvision_few_shot_dataset.jsonl` contains the original RepairVision dataset.

### raw_outputs
This folder contains all raw outputs generated by the evaluated LLMs.
Each `.jsonl` file corresponds to a single evaluated model and dataset combination.  
The outputs contain:
- prompts,
- generated completions,
- token statistics,
- model identifiers,
- provider information.

The package contains outputs for all evaluated LLM families and model versions used throughout the thesis.

### evaluation_results
This folder contains the evaluated `.csv` results generated from the raw outputs.
The evaluation includes the metrics used throughout the thesis:
- Format Correctness,
- Structure Correctness,
- Change Structure Correctness,
- Type Structure Correctness,
- Type Isomorphism,
- Generated-in-Ground-Truth,
- Ground-Truth-in-Generated,
- At-Least-One-Correct-Edge.

### scripts
This folder contains all scripts required to reproduce the evaluation pipeline.
The folder includes:
- LLM querying clients,
- evaluation scripts,
- graph comparison utilities,
- mining utilities,
- parsing utilities,
- helper scripts used during the experiments.

## Environment Setup
The experiments were executed using Python 3.11.
Install dependencies using:
```bash
pip install -r requirements.txt
```

## Running the Evaluation Pipeline

To evaluate a model output file:
```bash
python scripts\paper_style_eval.py <input_jsonl> <output_csv> True
```

**Example:**
```bash
python scripts\paper_style_eval.py raw_outputs\results_gpt52_paper_synth.jsonl evaluation_results\gpt52_eval.csv True
```

## Running the LLM Pipeline

The LLM generation pipeline can be executed using:
```bash
python scripts\LLM_runner.py <provider> <model_id> <dataset> <output_file> SINGLE_EDGE
```

**Example:**
```bash
python scripts\LLM_runner.py openai gpt-5.2 synthetic_plus_custom.jsonl results_synthetic_plus_custom_gpt52.jsonl SINGLE_EDGE
```

## Notes

- API keys are not included in this replication package.
- Environment variables must be configured manually for provider access.
- The package contains only the generated outputs and evaluation artifacts required to reproduce the thesis results.
- The mining utilities included in the package are dependencies required for the graph-based evaluation pipeline.
